## 1. プロジェクトの目的

**shisan-app** は、資産運用の複利シミュレーションを行う単一ページWebアプリ。

- **現在(v1)**: 年齢・投資商品・配分額を入力すると、将来の資産推移を折れ線グラフで表示する
- **将来構想(v2以降)**: 日韓クロスボーダーの資産管理・相続シミュレーション機能へ拡張し、
  開発者(Lee)の独立後の商品ポートフォリオの一部とする
- **対象ユーザー**: 資産形成に関心のある個人。将来的には日韓にまたがる資産を持つ家族

**重要**: これは「学習プロジェクト」から「実用プロダクト候補」へ移行中。
今後の変更はすべて「他人に使ってもらえる品質か?」を基準に判断すること。

## 2. 技術構成

| 項目 | 内容 |
|---|---|
| フレームワーク | React 19.2 + Vite 8 |
| グラフ | Recharts 3.9 |
| Lint | oxlint(`.oxlintrc.json`) |
| 言語 | JavaScript(.jsx)— TypeScript化はロードマップP2で段階導入 |
| テスト | Vitest 4(`npm test`)— まず複利計算ロジックの単体テストを整備済み |
| デプロイ | Vercel想定(未確認 — ロードマップP3で確定) |
| リポジトリ | github.com/hayabuta737/shisan-app |

### 主要ファイル
- `src/App.jsx` — アプリ本体(入力フォーム、商品選択、グラフ描画、バリデーション)
- `src/lib/simulation.js` — 複利計算ロジック(UI非依存の純関数。テスト対象)
- `src/lib/simulation.test.js` — simulation.js の単体テスト(Vitest)
- `src/App.css` — ネイビー×ゴールドの高級感デザイン(このトーンを維持すること)
- `src/index.css` — グローバル変数・ベーススタイル

### コマンド
```bash
npm run dev      # 開発サーバー起動
npm run build    # 本番ビルド
npm run lint     # oxlintによる静的チェック
npm test         # Vitestによる単体テスト
```

## 3. 絶対に守るルール

1. **金融データの信頼性が最優先**
   - 商品の期待利回りは必ず出典(参考にした指数・期間)をコードコメントに明記する
   - 冗談・プレースホルダーの値を商品データに入れない
   - 免責事項の表示を削除・縮小しない
2. **計算ロジックの変更は検算とセット**
   - 複利計算を変更したら、手計算で検証できる具体例(例: 100万円・年利5%・10年 → 約162.9万円)をコメントまたはテストに残す
3. **変更は小さく、1タスク1コミット**
   - 大きなリファクタリングと機能追加を同じコミットに混ぜない
4. **既存のUIデザイン(ネイビー×ゴールド)のトーンを壊さない**
5. **ドキュメントの同期**
   - 仕様を変更したら、このCLAUDE.mdの該当箇所とREADMEを同じ作業内で更新する
   - (過去に仕様書と実装がズレた「設計ドリフト」が発生した教訓)
