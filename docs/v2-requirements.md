作成日: 2026-07-12
この文書は v2 開発の唯一の正式仕様である。実装がこの文書とズレる場合、実装前に必ず指摘すること。

## 0. アプリの目的(すべての判断基準)

20〜40代の若手・中堅世代が、
**「自分が生涯どれくらいお金が必要か」を知り、
「time in the market(市場に居続ける)という正しい資産運用で、それが無理なく達成可能」だと理解する**ことで、
お金稼ぎという手段への没頭から解放され、人生そのものを楽しめるようになること。

- このアプリの敵は「無知」ではなく「不安」。不安は数字の不在から生まれる
- 判断に迷ったら「ユーザーの不安が減る方」を選ぶこと
- 誠実さ(現実的な数値・不確実性の明示)を演出より優先すること

## 2. 画面構成:3ステップの一本道

v1とv2をタブで並べず、以下の1フローに統合する。

```
ステップ1「必要額を知る」 → ステップ2「運用を選ぶ」 → ステップ3「自由になる日を見る」
```

- **ステップ1**: 生涯必要額の入力フォーム(下記3章)
- **ステップ2**: 旧v1の商品選択UIをここに移設。商品構成から期待利回りを算出
  (旧v1の機能はここに吸収し、独立タブとしては残さない)
- **ステップ3**: グラフ+サマリーカード(下記5〜7章)

モバイル(スマホ)表示を最優先で設計すること。

## 3. ⑴ 生涯必要額の計算(フォーム方式)

### 入力項目
| 項目 | 入力方法 | 未入力時の補完 |
|---|---|---|
| 現在の年齢 | 数値 | 必須入力 |
| 基本生活費(月額・家賃/ローン含む) | 数値 | 政府データ(総務省・家計調査の世帯人数別平均) |
| 寿命想定 | 選択(90/95/100歳) | 100歳(厚労省・簡易生命表を参考表示) |
| 子どもの人数 | 0〜4 | 0 |
| 教育方針 | 公立中心/私立中心 | 公立中心(文科省・教育費調査等) |
| リタイア年齢 | 数値 | 65歳 |
| 年金の見込み月額 | 数値・手入力 | 「ねんきん定期便の見方」ヒントを表示し入力を促す。未入力時は厚労省モデル年金額 |
| その他の予定(自由記入) | 項目名+金額+年齢を複数行追加可能 | なし |

### 政府公式データの扱い(重要)
- `src/data/officialData.js` に定数として埋め込む
- **すべての数値に出典コメント必須**: 統計名・調査年・URL
- 実在しない統計値を創作しないこと。確認できない場合はその旨を報告し、ユーザーに確認を求める
- データ更新日を定数ファイル冒頭に明記(年1回の手動更新を想定)

### 計算の考え方
- 生涯必要額 = 現在から寿命までの(生活費+教育費+自由記入イベント) − (リタイア後の年金収入合計)
- インフレ率 年2% を全支出に適用(定数として変更可能に)

## 4. ⑵ 必要な毎月積立額の算出

- ユーザー入力(+政府データ補完)から、**「毎月○○円」という1つの数字**を算出して提示する
- 計算: 標準シナリオ(下記6章)の利回りで、寿命時点まで資産が枯渇しない最小の毎月積立額を求める
- ユーザーが自分の積立額を別途入力する機能は付けない(v2.0では提示のみ)

---

## 5. ⑶ 安心ライン(クロスオーバーポイント)

- グラフに2本の線を引く:
  1. **資産曲線**(右肩上がり): 毎月積立+複利運用による資産推移
  2. **必要額ライン(右肩下がり)**: 「その時点から寿命までに必要な残額」
     ※生涯総額の水平線にしないこと(金融的に不正確なため)
- 2本が交差する点 = クロスオーバーポイント
- 交差時点を「**あなたがお金の心配から自由になる日: 20XX年(XX歳)**」と目立つ形で表示
  (年次シミュレーションのため月の粒度は算出せず、年+年齢で表示する)
- 交差しない場合は建設的に表示(不安を煽らない)
  ※v2.0の実装補足: 表示する積立額は常に「寿命まで到達可能な最小額」のため、
  交差しないのは積立期間が取れない入力(既にリタイア済み等)に限られる。
  その場合は「リタイア年齢や生活費など条件の見直し」を促す文言を表示する
  (当初案の「積立額を月+○円にすると到達」は、ユーザーが自分の積立額を
  入力しないv2.0の設計では発生しないため適用外)

---

## 6. ⑷ シナリオ比較

- 資産曲線を3本表示: 悲観(年2%)/標準(年4%)/楽観(年6%)を初期値とする
- ステップ2の商品構成から標準利回りを算出する場合も、悲観・楽観は±2%の幅で描く
- 凡例に「将来は不確実です。幅で考えることが大切です」の一文を添える
- 各利回りの根拠(参考にした長期実績)をコメントに明記

---

## 7. ⑸ 結果サマリーカード

グラフの上部に3枚のカードを大きな数字で表示:
1. **生涯必要額**(例: 2.4億円)
2. **そのための毎月積立額**(例: 月4.2万円)
3. **運用で増えた分**(寿命時点までの累計運用益。標準シナリオ基準)

- 数字は一瞬で読める大きさ・桁区切り・単位(万円/億円)自動変換
- カードの下に補足一文(例: 「時間があなたの味方です」)

## 8. 技術要件

- 計算ロジックはすべて `src/lib/lifeplan.js` に分離(UIから独立させる)
- **Vitest導入。計算ロジックに手計算で検証済みの単体テストを最低8ケース**
  (例: 生活費のみ/子ども2人私立/年金多め/交差しないケース等)
- 既存のCLAUDE.mdの「絶対に守るルール」をすべて適用
- デザインは既存のネイビー×ゴールドのトーンを踏襲
- 開発は `v2-lifeplan` ブランチで行い、完成後にmainへ統合
